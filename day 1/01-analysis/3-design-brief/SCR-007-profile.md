## 1. Назначение и контекст

Экран показывает данные клиента и дает базовые действия по управлению сессией и профилем.

---

## 2. Структура и иерархия

- Имя клиента.
- Телефон.
- Данные профиля.
- Действие выхода из аккаунта.

---

## 3. Состояния экрана

| Состояние | Поведение |
|-----------|-----------|
| Content | Профиль загружен |
| Loading | Данные профиля загружаются |
| Error | Ошибка при загрузке или обновлении |

---

## 4. Поведение

- Профиль должен показывать только данные текущего клиента.
- Экран поддерживает выход из аккаунта.
- При выходе пользователь возвращается к экрану авторизации.

## 6. Критерии приемки

```gherkin
AC-001: Просмотр профиля
  Дано клиент авторизован
  Когда он открывает профиль
  Тогда видит свои данные

AC-002: Выход из аккаунта
  Дано клиент находится в профиле
  Когда он выходит из аккаунта
  Тогда система возвращает его на экран входа
```
