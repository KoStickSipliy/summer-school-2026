## 1. Назначение и контекст

Экран показывает все записи клиента: предстоящие, завершенные и отмененные. Он нужен для контроля истории и быстрого перехода к деталям брони.

---

## 2. Структура и иерархия

- Заголовок экрана.
- Список записей.
- Секция активных и завершенных бронирований.
- Переход в детали конкретной записи.

---

## 3. Состояния экрана

| Состояние | Поведение |
|-----------|-----------|
| Loading | Скелетоны списка записей |
| Content | Показаны все записи клиента |
| Empty | Нет записей, показана заглушка |
| Error | Ошибка загрузки и кнопка повторить |

---

## 4. Поведение

- Клиент видит только свои записи.
- У каждой записи должен быть понятный статус.
- Из списка можно перейти в детали брони.

## 6. Критерии приемки

```gherkin
AC-001: Список записей
  Дано у клиента есть записи
  Когда экран загружен
  Тогда отображается список его бронирований

AC-002: Переход в детали
  Дано на экране показана запись
  Когда пользователь нажимает на нее
  Тогда открываются детали брони
```
