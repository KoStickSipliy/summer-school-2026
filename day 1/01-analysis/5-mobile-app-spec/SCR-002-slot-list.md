**ID:** SCR-002 · **Тип:** Экран · **Зона:** АЗ · **Приоритет:** Critical · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-04

## 1. Назначение и контекст

SCR-002 — главный экран авторизованной зоны. Он показывает доступные слоты на ближайшие 7 дней и даёт вход в фильтры, карточку слота и другие основные разделы.

## 2. Пользователь и цель

**Роль — Клиент.**

**Цель на экране:** быстро найти подходящее занятие и открыть его детали.

## 4. Входные данные

| Название | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| `date_from` | Фильтр | datetime | Начало диапазона просмотра слотов |
| `date_to` | Фильтр | datetime | Конец диапазона просмотра слотов |
| `route_type` | Фильтр | `novice`, `experienced` | Тип маршрута |
| `instructor_id` | Фильтр | UUID | Мастер |
| `only_available` | Фильтр | `true` / `false` | Только слоты со свободными местами |

## 5. Используемые запросы

### GET /slots

**Триггер:** при открытии экрана и при изменении фильтров.

**Назначение:** получить список слотов.

**Параметры:** `date_from`, `date_to`, `route_type`, `instructor_id`, `only_available`, `limit`, `offset`.

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| `200` и `items` не пуст | Показать список |
| `200` и `items` пуст | Показать empty state |
| `400` / `401` | Показать ошибку и возможность обновить |
| `5xx` | Показать error state |

### GET /instructors

**Триггер:** загрузка фильтров по мастеру.

**Назначение:** получить справочник мастеров для выбора в фильтрах.

## 6. Состояния экрана

| Состояние | Поведение |
|-----------|-----------|
| Loading | Скелетоны карточек слотов |
| Content | Список доступных слотов |
| Empty | Понятная заглушка без доступных слотов |
| Empty by filters | Подсказка изменить или сбросить фильтры |
| Error | Сообщение и кнопка обновления |

## 7. Поведение

- По умолчанию показываются слоты на ближайшие 7 дней.
- Карточка слота является единой тап-зоной.
- Слоты без свободных мест показываются как недоступные для записи.
- Список можно обновить pull-to-refresh.

## 9. Критерии приемки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** клиент открыл список слотов, **Когда** данные загружены, **Тогда** видны слоты на ближайшие 7 дней |
| AC-002 | **Дано** применён хотя бы один фильтр, **Когда** список отображён, **Тогда** в хедере виден индикатор активных фильтров |
| AC-003 | **Дано** по фильтрам нет результатов, **Когда** список обновлён, **Тогда** показана заглушка с подсказкой изменить или сбросить фильтры |
