# Требования на дизайн · SCR-001 · Регистрация / Вход

**ID:** SCR-001 · **Тип:** Экран · **Зона:** НЗ · **Приоритет:** Critical · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-04

**Источники:** [Бизнес-требования](../2-requirements/business-requirements.md) · [Функциональные требования](../2-requirements/functional-requirements.md) · [Use cases](../2-requirements/use-cases.md) · [User stories](../2-requirements/user-stories.md)

## 1. Назначение и контекст

SCR-001 — точка входа в приложение. Экран должен поддерживать вход по номеру телефона с подтверждением OTP и первичную регистрацию без пароля.

## 2. Пользователь и цель

**Роль — Клиент.**

**Цель на экране:** получить доступ к списку слотов и сохранить имя в профиле при первом входе.

## 3. Навигация

### Входящие переходы
- Старт приложения без активной сессии.
- Выход из аккаунта с [SCR-007-profile.md](SCR-007-profile.md).

### Исходящие переходы
- Успешная авторизация → [SCR-002-slot-list.md](SCR-002-slot-list.md).

## 4. Входные данные

| Название | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| `phone` | Состояние формы | `+79991234567` | Номер телефона в формате E.164 |
| `code` | Состояние формы | `1234` .. `123456` | OTP-код из SMS |
| `name` | Состояние формы | Строка | Имя клиента при первичной регистрации |
| `is_new` | Ответ API | `true` / `false` | Флаг новой регистрации |

## 5. Используемые запросы

### POST /auth/request-code

**Триггер:** пользователь нажимает «Получить код».

**Назначение:** отправить OTP на номер телефона.

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| `200` | Показать таймер повторной отправки кода |
| `400` | Показать ошибку валидации телефона |
| `429` | Показать сообщение о лимите повторных запросов |
| `5xx` | Показать ошибку с возможностью повторить |

### POST /auth/verify-code

**Триггер:** пользователь подтверждает код.

**Назначение:** проверить OTP и получить токены сессии.

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| `200` | Сохранить токены, клиента и перейти к слоту списку |
| `400` | Показать ошибку кода или телефона |
| `429` | Показать лимит попыток |
| `5xx` | Показать ошибку с повтором |

## 6. Состояния экрана

| Состояние | Поведение |
|-----------|-----------|
| Default | Показано поле телефона и кнопка запроса кода |
| Code requested | Показан ввод OTP и таймер повторной отправки |
| New client | Дополнительно показано поле имени |
| Loading | Кнопка заблокирована, отображается лоадер |
| Error | Показано понятное сообщение и повтор действия |

## 7. Действия пользователя

| Действие | Результат |
|----------|-----------|
| Ввод телефона и запрос кода | Отправка `POST /auth/request-code` |
| Ввод OTP | Отправка `POST /auth/verify-code` |
| Ввод имени для нового клиента | Имя передается в `verify-code` |

## 9. Критерии приемки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** клиент открыл приложение без сессии, **Когда** он вводит номер телефона и подтверждает OTP, **Тогда** система авторизует клиента и открывает список слотов |
| AC-002 | **Дано** клиент входит впервые, **Когда** он указывает имя при подтверждении кода, **Тогда** имя сохраняется в профиле |
| AC-003 | **Дано** пользователь ошибся в телефоне или коде, **Когда** сервер возвращает 400, **Тогда** экран показывает понятную ошибку и не теряет введённые данные |
