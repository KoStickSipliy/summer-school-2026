# Требования на дизайн · SCR-005 · Мои бронирования

**ID:** SCR-005 · **Тип:** Экран · **Зона:** АЗ · **Приоритет:** Critical · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-04

**Источники:** [Функциональные требования](../2-requirements/functional-requirements.md) · [Use cases](../2-requirements/use-cases.md) · [User stories](../2-requirements/user-stories.md)

## 1. Назначение и контекст

Экран показывает все бронирования клиента: предстоящие, завершённые и отменённые. Он нужен для контроля истории и быстрого перехода к деталям брони.

## 2. Навигация

### Входящие переходы
- Тап по табу «Мои бронирования».
- Возврат из [SCR-004-booking.md](SCR-004-booking.md) после успешного бронирования.

### Исходящие переходы
- Тап по записи → [SCR-006-booking-details.md](SCR-006-booking-details.md).

## 3. Входные данные

| Название | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| `status` | Фильтр API | `active`, `cancelled`, `late_cancel`, `club_cancelled` | Фильтр списка броней |
| `limit` | Параметр API | Integer | Размер страницы |
| `offset` | Параметр API | Integer | Смещение |

## 4. Используемые запросы

### GET /bookings

**Триггер:** открытие экрана и обновление списка.

**Назначение:** получить список бронирований текущего клиента.

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| `200` и `items` не пуст | Показать список |
| `200` и `items` пуст | Показать empty state |
| `401` | Вернуть на вход |
| `5xx` | Показать error state |

## 5. Состояния экрана

| Состояние | Поведение |
|-----------|-----------|
| Loading | Скелетоны списка записей |
| Content | Показаны все записи клиента |
| Empty | Нет записей, показана заглушка |
| Error | Ошибка загрузки и кнопка повторить |

## 6. Поведение

- Клиент видит только свои записи.
- У каждой записи должен быть понятный статус.
- Из списка можно перейти в детали брони.
- Причина отмены отображается, если она пришла от API.

## 8. Критерии приемки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** у клиента есть записи, **Когда** экран загружен, **Тогда** отображается список его бронирований |
| AC-002 | **Дано** на экране показана запись, **Когда** пользователь нажимает на неё, **Тогда** открываются детали брони |
