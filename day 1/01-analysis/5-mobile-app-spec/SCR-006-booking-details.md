# Требования на дизайн · SCR-006 · Детали брони + отмена

**ID:** SCR-006 · **Тип:** Экран · **Зона:** АЗ · **Приоритет:** Critical · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-04

**Источники:** [Бизнес-требования](../2-requirements/business-requirements.md) · [Функциональные требования](../2-requirements/functional-requirements.md) · [Use cases](../2-requirements/use-cases.md)

## 1. Назначение и контекст

Экран показывает полную информацию о записи и даёт возможность отменить её в допустимый срок.

## 2. Навигация

### Входящие переходы
- Тап по записи в [SCR-005-my-bookings.md](SCR-005-my-bookings.md).

### Исходящие переходы
- Тап по отмене → [BS-003-cancel-confirm.md](BS-003-cancel-confirm.md).
- Тап по карте маршрута → [BS-004-route-map.md](BS-004-route-map.md).

## 3. Входные данные

| Название | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| `booking_id` | Параметр перехода | UUID | Идентификатор записи |
| `booking` | Ответ API | Booking | Полные данные брони |

## 4. Используемые запросы

### GET /bookings/{bookingId}

**Триггер:** открытие экрана.

**Назначение:** получить детальную информацию о брони со вложенными данными слота.

### POST /bookings/{bookingId}/cancel

**Триггер:** подтверждение отмены.

**Назначение:** отменить бронь, если действие доступно по времени.

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| `200` | Обновить статус брони и детали |
| `409` | Показать, что бронь уже отменена |
| `422` | Показать, что отмена недоступна после начала занятия |
| `403` / `404` | Показать, что бронь недоступна |

## 5. Состояния экрана

| Состояние | Поведение |
|-----------|-----------|
| Content | Подробности брони показаны полностью |
| Cancel available | Кнопка отмены активна |
| Cancel unavailable | Кнопка скрыта или недоступна |
| Error | Ошибка загрузки данных |

## 6. Правила отмены

- Отменить запись можно только не позднее чем за 1 час до начала занятия.
- Если срок прошёл, система должна показать недоступность отмены.
- После отмены статус в списке и деталях должен обновиться.
- Причина отмены от мастерской должна отображаться, если она есть в ответе API.

## 8. Критерии приемки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** до начала занятия осталось больше 1 часа, **Когда** пользователь открывает детали брони, **Тогда** он видит возможность отменить запись |
| AC-002 | **Дано** до начала занятия осталось меньше 1 часа, **Когда** пользователь пытается отменить запись, **Тогда** система сообщает, что отмена недоступна |
| AC-003 | **Дано** бронь отменена, **Когда** экран обновлён, **Тогда** статус в списке и деталях соответствует отмене |
