# Требования на дизайн · SCR-007 · Профиль клиента

**ID:** SCR-007 · **Тип:** Экран · **Зона:** АЗ · **Приоритет:** Medium · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-04

**Источники:** [Функциональные требования](../2-requirements/functional-requirements.md) · [Бизнес-требования](../2-requirements/business-requirements.md)

## 1. Назначение и контекст

Экран показывает данные клиента и даёт базовые действия по управлению сессией и профилем.

## 2. Навигация

### Входящие переходы
- Тап по табу «Профиль».

### Исходящие переходы
- Выход из аккаунта → [SCR-001-registration.md](SCR-001-registration.md).

## 3. Входные данные

| Название | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| `client` | Ответ API | Client | Текущие данные клиента |
| `name` | Поле формы | Строка | Новое имя клиента при редактировании профиля |

## 4. Используемые запросы

### GET /profile

**Триггер:** открытие экрана.

**Назначение:** получить данные текущего клиента.

### PATCH /profile

**Триггер:** сохранение нового имени.

**Назначение:** обновить имя клиента.

### POST /auth/logout

**Триггер:** выход из аккаунта.

**Назначение:** завершить текущую сессию.

## 5. Состояния экрана

| Состояние | Поведение |
|-----------|-----------|
| Content | Профиль загружен |
| Loading | Данные профиля загружаются |
| Error | Ошибка при загрузке или обновлении |

## 6. Поведение

- Профиль должен показывать только данные текущего клиента.
- Экран поддерживает выход из аккаунта.
- При выходе пользователь возвращается к экрану авторизации.
- Имя клиента может быть изменено через API профиля.

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** клиент авторизован, **Когда** он открывает профиль, **Тогда** видит свои данные |
| AC-002 | **Дано** клиент находится в профиле, **Когда** он выходит из аккаунта, **Тогда** система возвращает его на экран входа |
