# Требования на дизайн · SCR-004 · Оформление записи

**ID:** SCR-004 · **Тип:** Экран · **Зона:** АЗ · **Приоритет:** Critical · **Статус:** Черновик · **Версия:** 0.1 · **Дата:** 2026-07-04

**Источники:** [Бизнес-требования](../2-requirements/business-requirements.md) · [Функциональные требования](../2-requirements/functional-requirements.md) · [Use cases](../2-requirements/use-cases.md) · [User stories](../2-requirements/user-stories.md)

## 1. Назначение и контекст

Экран собирает параметры бронирования и завершает сценарий записи. Здесь клиент подтверждает действие и видит итоговую стоимость до отправки запроса.

## 2. Навигация

### Входящие переходы
- Из [SCR-003-slot-card.md](SCR-003-slot-card.md).

### Исходящие переходы
- Успешное бронирование → [BS-002-booking-success.md](BS-002-booking-success.md).

## 3. Входные данные

| Название | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| `slot_id` | Параметр перехода | UUID | Слот, на который создаётся бронь |
| `equipment_mode` | Состояние формы | `own`, `rental` | Свои инструменты и фартук или прокат от мастерской |
| `price_total` | Предварительный расчёт | Integer | Итоговая стоимость до подтверждения |

## 4. Используемые запросы

### POST /bookings

**Триггер:** пользователь нажимает кнопку подтверждения.

**Обязательные данные:** `slot_id`, `equipment_mode`, `Idempotency-Key`.

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| `201` | Показать экран успеха и сохранить бронь в списке |
| `409` | Показать отказ в записи, если мест уже нет |
| `410` | Показать, что слот отменён мастерской |
| `422` | Показать семантическую ошибку |
| `400` / `401` / `5xx` | Показать ошибку и возможность повторить |

## 5. Состояния экрана

| Состояние | Поведение |
|-----------|-----------|
| Editing | Пользователь выбирает параметры записи |
| Loading | Отправляется запрос на бронирование |
| Error | Показывается понятная ошибка с возможностью повторить действие |

## 6. Правила и ограничения

- Пользователь записывается только на себя.
- Стоимость записи показывается до подтверждения.
- Если мест больше нет, бронь не создается.
- После успешной записи приложение должно показать подтверждение.
- Сценарий должен корректно обрабатывать ошибки API.

## 8. Критерии приемки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** клиент выбрал параметры записи, **Когда** он видит экран подтверждения, **Тогда** отображается итоговая стоимость до отправки запроса |
| AC-002 | **Дано** параметры записи валидны, **Когда** клиент подтверждает запись, **Тогда** система отправляет запрос в бэкенд и показывает подтверждение |
| AC-003 | **Дано** свободных мест больше нет, **Когда** клиент пытается записаться, **Тогда** система показывает отказ в записи |
