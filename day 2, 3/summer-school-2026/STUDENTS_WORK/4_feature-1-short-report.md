# Краткий отчёт по фиче 1 (SCR-002)

## Что реализовано
- Добавлен `refresh`-сценарий в `SlotListStore`: обновление поверх текущего контента без сброса в `Error`.
- Добавлено сообщение при неуспешном обновлении: «Не удалось обновить. Проверьте соединение и попробуйте снова.»
- Для загрузки списка добавлено дефолтное окно дат `now .. now + 7 дней`.
- Добавлена сортировка слотов по времени старта (ближайшие выше).
- В `SlotListScreen` добавлены:
  - `pull-to-refresh`,
  - индикатор количества активных фильтров,
  - группировка слотов по дням (day headers),
  - корректная блокировка недоступных слотов (`freeSeats == 0` или `status=cancelled`).

## Тесты
- Добавлены unit-тесты: `SlotListStoreTest`:
  - дефолтный диапазон 7 дней и сортировка,
  - refresh с сохранением контента при ошибке,
  - `EmptyReason.NoSlotsByFilters` при пустой выдаче с фильтрами.
- Прогон:
  - `:shared:wasmJsTest --rerun-tasks` — успешно.
